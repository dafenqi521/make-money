# 场内 ETF 趋势轮动开发与验证步骤

## 当前已完成

### 1. 独立扫描与选基

- `src/strategy/etf_rotation.py`：过滤、评分、相关性去重、目标权重和退出规则；
- `src/engine/rotation_scanner.py`：默认或自定义ETF池并发读取、错误隔离和扫描日期；
- `src/engine/etf_universe.py`：全市场ETF目录自动发现、基础排除、质量报告和快照回退；
- 页面显示每只ETF的因子、排名和淘汰原因。

### 2. 扫描驱动的模拟调仓

- `src/engine/paper_trading.py`：目标组合与当前持仓对比；
- 生成买入、卖出、持有、当前份额、目标份额和原因；
- 卖出先于买入，按100份取整；
- 模拟成交支持佣金、最低佣金和可配置滑点；
- 行情过期、扫描覆盖率不足或持仓数据缺失时冻结交易。
- 页面即时预扫高流动性前80只；后台任务扫描全部通过流动性预筛的ETF；
- 下一开市日09:35–10:00首选确认，确认时按当日买一/卖一价复算订单；
- 法定休市、30秒陈旧盘口、盘口深度不足、涨跌停或价格偏离超过3%时冻结；
- 唯一信号批次和数据库执行锁阻止重复成交。

### 3. 本地模拟账户

- `src/engine/portfolio.py`：现金、持仓、平均成本、盈亏、成交和统一T+1；
- 保存建仓日期、最高价、弱势确认天数和最后信号日期；
- 支持整张调仓清单模拟执行和单笔手工成交登记。

### 4. 单账户持久化和绩效

- `src/data/portfolio_db.py`：默认SQLite，配置`DATABASE_URL`后切换PostgreSQL；
- 项目只保存一个模拟账户，不提供多人登录或账户切换；
- 每日净值快照、累计收益、最大回撤和卖出胜率；
- JSON账户备份和恢复；
- 净值累计8个日快照后展示折线趋势。

### 5. Streamlit一体化页面

页面顺序为：

1. 账户总资产、收益、回撤、现金、持仓和胜率；
2. 模拟账户净值趋势；
3. 行情日期、扫描覆盖率和目标组合；
4. 调仓清单与批量模拟执行；
5. 当前持仓、手工成交和交易记录；
6. 全部候选与数据错误；
7. JSON备份恢复和账户重置。

### 6. 组合历史回测

- `src/engine/backtest.py`逐交易日回放唯一ETF轮动策略；
- T日收盘生成信号，T+1交易日开盘加入不利滑点后模拟成交；
- 执行佣金、最低佣金、100份整数、现金约束和统一T+1；
- 输出策略/基准净值、年化收益、波动率、夏普、最大回撤、卡玛、换手和平均资金暴露；
- 展示回撤路径、月度收益和成交明细；
- 支持最多持有2/3/4/5只的一维参数稳健性检查，避免组合式挑选历史最优值；
- 下载失败和低于80%的数据覆盖率均显式展示。

## 下一阶段：连续模拟盘验证

1. 每个交易日收盘后或次日确认前打开页面，检查自动扫描结果；
2. 保存当日JSON备份；
3. 次日09:35–10:00确认模拟清单，真实手工交易则登记实际成交价；
4. 连续运行至少20个交易日；
5. 每周核对现金、持仓市值、成交费用和账户总资产；
6. 重点观察节假日数据新鲜度、跨境ETF异常价格、停牌和网络失败；
7. 检查是否存在重复成交、T+1误卖或因候选池修改导致的持仓遗漏。

## 后台自动化

- GitHub Actions在交易日北京时间15:20运行目录刷新和收盘扫描；
- 09:35检查待确认批次并调用可选`NOTIFY_WEBHOOK_URL`；
- 交易所日历负责节假日跳过；
- 任务只生成建议和提醒，不连接券商；
- 必须在GitHub Actions和Streamlit中配置同一个PostgreSQL `DATABASE_URL`。

## 下一阶段可选开发

### 跨境ETF折溢价门禁

当前仍未接入实时IOPV。正式把跨境/QDII ETF用于真实交易参考前，应增加IOPV、境外市场休市和高溢价公告门禁，或者先关闭海外权益类别。

### 回测数据进一步严谨化

数据库现在保存最新有效目录快照，但回测仍使用“今天的目录”向历史回溯，仍可能存在幸存者偏差。后续应保留逐日目录历史、基金规模、停牌和退市状态，并增加滚动样本外验证。

### 真实券商接口

当前版本明确不连接券商。任何真实接口都必须单独设计认证、权限、成交回报、撤单、风控熔断和人工确认安全门，不能直接复用模拟执行按钮。
