# 场内 ETF 趋势轮动开发与验证步骤

## 当前已完成

### 1. 独立扫描与选基

- `src/strategy/etf_rotation.py`：过滤、评分、相关性去重、目标权重和退出规则；
- `src/engine/rotation_scanner.py`：默认或自定义ETF池并发读取、错误隔离和扫描日期；
- 页面显示每只ETF的因子、排名和淘汰原因。

### 2. 扫描驱动的模拟调仓

- `src/engine/paper_trading.py`：目标组合与当前持仓对比；
- 生成买入、卖出、持有、当前份额、目标份额和原因；
- 卖出先于买入，按100份取整；
- 模拟成交支持佣金、最低佣金和可配置滑点；
- 行情过期、扫描覆盖率不足或持仓数据缺失时冻结交易。

### 3. 本地模拟账户

- `src/engine/portfolio.py`：现金、持仓、平均成本、盈亏、成交和统一T+1；
- 保存建仓日期、最高价、弱势确认天数和最后信号日期；
- 支持整张调仓清单模拟执行和单笔手工成交登记。

### 4. 持久化和绩效

- `src/data/portfolio_db.py`：SQLite账户状态、持仓和成交；
- 每日净值快照、累计收益、最大回撤和卖出胜率；
- JSON账户备份和恢复；
- 净值累计8个日快照后展示折线趋势。

### 5. Streamlit一体化页面

页面顺序为：

1. 账户总资产、收益、回撤、现金、持仓和胜率；
2. 模拟账户净值趋势；
3. 行情日期、扫描覆盖率和目标组合；
4. 调仓清单与批量模拟执行；
5. 当前持仓、手工成交和交易记录；
6. 全部候选与数据错误；
7. JSON备份恢复和账户重置。

## 下一阶段：连续模拟盘验证

1. 每个交易日收盘后运行一次扫描；
2. 保存当日JSON备份；
3. 次日按模拟清单执行，真实手工交易则登记实际成交价；
4. 连续运行至少20个交易日；
5. 每周核对现金、持仓市值、成交费用和账户总资产；
6. 重点观察节假日数据新鲜度、跨境ETF异常价格、停牌和网络失败；
7. 检查是否存在重复成交、T+1误卖或因候选池修改导致的持仓遗漏。

## 后续可选开发

### 组合历史回测

当前模拟账户用于向前验证，不等同于完整历史回测。严谨历史回测应新增组合级引擎：

1. 每个交易日只把截至T-1的数据传给选基内核；
2. T日使用可成交价格执行先卖后买；
3. 保留历史时点ETF名单，控制幸存者偏差；
4. 记录每日现金、持仓、市值、目标权重和未成交原因；
5. 输出收益、回撤、波动、换手、持有期和成本敏感性；
6. 用合成数据优先验证无未来数据、100份取整和T+1。

### 持久化升级

如果Streamlit部署需要多用户或长期可靠运行，应将SQLite替换为带身份隔离的外部数据库，并增加认证。公开云端SQLite不适合作为多用户真实账户系统。

### 真实券商接口

当前版本明确不连接券商。任何真实接口都必须单独设计认证、权限、成交回报、撤单、风控熔断和人工确认安全门，不能直接复用模拟执行按钮。
