# 场内 ETF 多资产趋势轮动策略 V1

## 1. 策略结论

策略采用“绝对趋势过滤 + 横截面动量排名 + 多资产去重 + 波动率仓位 + 分层退出”。项目自己的行情扫描结果是唯一信号源，不依赖聚宽或其他托管量化平台。

策略优先控制回撤、数据风险和无效换手，不以短期暴利为目标。任何历史或模拟结果都不代表未来收益。

## 2. 交易边界

- 标的：本项目成功读取的沪深交易所场内 ETF；
- 默认池：19只代表性宽基、行业、海外、商品和债券ETF；
- 自定义池：用户输入的有效6位场内ETF代码；
- 暂不包含：LOF、封闭式基金、REIT、货币ETF和一级申购赎回；
- 暂不使用：融资、杠杆、反向产品和折溢价套利；
- 交易单位：100份整数；
- 统一T+1：即使个别ETF支持T+0，策略也不卖出当日新买份额；
- 模拟费用：佣金双边万三、单笔最低5元、不计股票印花税、默认单边滑点0.10%。

## 3. 候选池与数据门禁

项目只对本次扫描成功的ETF排名。以下情况不会被转换成买卖信号：

1. 单只ETF历史行情读取失败：冻结该持仓的自动交易；
2. 全部候选扫描成功率低于80%：冻结全部自动调仓；
3. 最新完整日线距离模拟成交日超过7个自然日：冻结全部自动调仓；
4. 参考价格无效：冻结该标的；
5. 确认时当日实时盘口覆盖率低于80%：冻结全部模拟调仓；
6. 无排名结果：不生成调仓清单。

当日动态日线在北京时间15:05前不参与排名，避免盘中未完成K线形成前视信号。

ETF硬过滤条件：

1. 上市不少于120个自然日；
2. 至少有121根完整日线；
3. 最近20日平均成交额不低于3000万元；
4. 最近20日任一日成交额不低于500万元；
5. 名称不包含货币、现金、理财、分级、杠杆、反向或REIT；
6. 最新收盘价 > MA20 > MA60；
7. MA5 > MA10；
8. 20日收益率为正。

## 4. 评分与选基

所有因子都转为候选池内百分位排名：

```text
综合分 =
    35% × 20日收益率百分位
  + 30% × 60日收益率百分位
  + 15% × 120日收益率百分位
  + 10% × 低20日波动率百分位
  + 10% × 低60日最大回撤百分位
```

从高分到低分选择，并执行两层去重：

- 资产类别上限：境内宽基2、境内行业1、海外权益1、商品1、债券1、其他1；
- 任意候选与已选ETF的60日收益相关系数达到0.90时跳过。

最终最多持有4只。

## 5. 仓位与调仓

- 保留至少10%现金；
- 对入选ETF使用20日波动率倒数加权；
- 单只ETF权重不超过30%；
- 行业主题不超过25%，海外权益不超过30%，商品不超过25%，债券不超过40%；
- 权重受限后的剩余资金保留为现金；
- 目标金额按模拟账户总资产计算；
- 目标份额按100份向下取整；
- 执行顺序始终为先卖后买；
- 现金不足时，买单按可负担的100份整数缩减。

## 6. 买入规则

只有同时满足以下条件才生成模拟买入：

1. ETF位于本次目标组合；
2. 数据质量门禁全部通过；
3. 目标份额大于当前份额；
4. 至少可以买入100份并支付佣金；
5. 完整日线收盘价只用于生成信号；确认时买入优先使用当日卖一价、卖出优先使用当日买一价，重新计算100份整数订单后再加入不利滑点。

当前版本按日线生成信号，不实现盘中VWAP或实时IOPV判断。实时盘口必须带有确认日日期；陈旧、缺失或覆盖率不足的报价不会用于模拟成交。真实交易仍需要用户在券商软件中确认价格和可成交性。

## 7. 卖出优先级

1. **动态硬止损**：亏损达到 `clip(2.5 × ATR20比例, 3%, 8%)`；
2. **移动止盈**：最高盈利达到2%后，从最高价回撤达到 `clip(2 × ATR20比例, 2.5%, 10%)`；
3. **趋势退出**：连续2次新日线收盘低于MA20，或MA5低于MA10；
4. **相对弱势退出**：持有满5个交易日后，连续2次跌出综合排名前8；
5. **时间止损**：持有满10个交易日、期间最高盈利不足1.5%，且排名低于合格池中位数；
6. **目标权重下调**：满足最短持有期后，将超出目标的部分减仓。

退出前必须通过数据质量门禁并检查T+1可卖份额。持仓不在本次成功扫描结果中时不自动卖出。

## 8. 模拟账户

- `PortfolioManager`记录现金、持仓、平均成本、最高价、弱势确认天数和成交；
- `PortfolioDB`默认保存到SQLite，配置`DATABASE_URL`后保存到PostgreSQL；
- 每个自然日保存最后一次账户净值快照；
- 累计至少8个日快照后展示净值趋势；
- 页面支持执行整张模拟调仓清单，也支持登记单笔手工成交；
- JSON备份包含账户、成交和净值快照，可用于云端重启后的恢复。

项目只使用一个模拟账户。SQLite适合本地单账户；Streamlit云端长期运行应配置PostgreSQL，JSON备份继续作为迁移和人工归档手段。

## 9. 历史回测口径

- 每个信号日只向策略提供截至当日收盘的历史数据；
- 调仓单在下一交易日开盘价基础上加入不利滑点后成交；
- 组合每日收盘估值，基准采用所选ETF的收盘价买入并持有；
- 费用包括双边佣金和最低佣金，不包含股票印花税；
- 数据覆盖率不足80%时，沿用实盘模拟的数据质量门禁冻结调仓；
- 结果展示累计/年化收益、波动、夏普、最大回撤、卡玛、换手、资金暴露和月度收益；
- 参数稳健性只做有限的一维比较，不自动挑选历史最优参数；
- 当前ETF池向历史回溯可能存在幸存者偏差，不能把回测收益视为未来收益预期。

## 10. 日常运行流程

```text
打开页面后自动扫描最新完整日线
→ 核对行情日期、成功率和失败标的
→ 查看目标组合与调仓清单
→ 下一开市日09:35–10:00用实时盘口复价
→ 用户确认模拟调仓或在券商手工交易
→ 将真实手工成交登记到模拟账户
→ 保存JSON备份并持续观察净值、回撤和成交记录
```

## 11. 验收标准

- 所有信号来自本项目成功读取的完整日线；
- 数据质量门禁失败时无自动成交；
- 非允许时段、同一信号日或当日盘口验证失败时无模拟成交；
- 所有订单符合100份整数和统一T+1约束；
- 模拟成交包含佣金、最低佣金和滑点；
- 目标组合、调仓清单、持仓和账户总资产能够对账；
- SQLite重启恢复和JSON备份恢复不丢失成交；
- PostgreSQL配置后重启能够恢复同一个模拟账户；
- 回测信号日晚于所用数据，成交日严格晚于信号日；
- 模拟账户连续运行至少20个交易日，无重复下单和状态丢失。

## 12. 风险边界

项目不会连接券商或自动提交真实订单。真实下单前，用户仍需核对交易所规则、ETF是否支持正常交易、盘口价格、券商佣金和账户可卖数量。

- [上海证券交易所 ETF 投教问答](https://etf.sse.com.cn/fund/quertion/)
- [中华人民共和国印花税法](https://fgk.chinatax.gov.cn/zcfgk/c100009/c5193058/content.html)
